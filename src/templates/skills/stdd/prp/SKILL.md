# STDD Skill: /stdd:prp

## Purpose
Generate structured planning document using What/Why/How/Success framework for clear requirement communication.

## When to Use
- Need to formalize a requirement before detailed specs
- Quick planning for small changes
- Communication artifact for stakeholder alignment

## Workflow
1. What: Describe the feature/change in clear terms
2. Why: Explain the business value and problem being solved
3. How: Outline the technical approach (high-level)
4. Success: Define acceptance criteria as checkable items
5. Output PRP document for review and confirmation

## Rules
- Keep each section concise (1-3 paragraphs max)
- Why must connect to business/user value
- How is high-level, not implementation details
- Success criteria must be checkable (checkboxes)

## Output
- PRP document with What/Why/How/Success sections
- Checkable acceptance criteria list
