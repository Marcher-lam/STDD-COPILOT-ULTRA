# STDD Skill: /stdd:vision

## Purpose
Create and maintain a project vision document that captures long-term goals, architectural north stars, and strategic direction.

## When to Use
- Starting a new project to define the vision
- Revisiting strategic direction after major changes
- Onboarding new team members
- Aligning implementation with business goals

## Workflow
1. Gather current project context, goals, and constraints
2. Create/update vision document covering:
   - Project purpose and problem statement
   - Target audience and use cases
   - Technical vision and architectural principles
   - Success metrics and milestones
   - Constraints and non-goals
3. Link vision to current specs and changes
4. Review consistency between vision and implementation

## Rules
- Vision should be stable but evolve with project
- Always check alignment: implementation should support vision
- Keep vision concise (1-2 pages)
- Include measurable success criteria

## Output
- stdd/vision.md - Project vision document
- Vision alignment report with current specs
